=========================================================================================================
BẢNG MAPPING KIỂU DỮ LIỆU: POSTGRESQL -> JAVA (SPRING DATA JPA CHO UNIPASS)
=========================================================================================================

POSTGRESQL TYPE       JAVA TYPE            IMPORT                         GHI CHÚ CHO JPA
---------------------------------------------------------------------------------------------------------
BIGSERIAL / BIGINT    Long                 java.lang.Long                 @Id, @GeneratedValue(strategy = GenerationType.IDENTITY)

INT                   Integer              java.lang.Integer

VARCHAR / CHAR        String               java.lang.String               

TEXT                  String               java.lang.String               Thêm annotation: @Column(columnDefinition = "TEXT")

BOOLEAN               Boolean              java.lang.Boolean              Dùng class Boolean (hỗ trợ null), không dùng kiểu nguyên thủy boolean

NUMERIC(p,s)          BigDecimal           java.math.BigDecimal           Tuyệt đối KHÔNG dùng Double/Float. Thêm: @Column(precision = 15, scale = 2)

TIMESTAMPTZ           Instant              java.time.Instant              Chuẩn UTC. Dùng kèm @CreationTimestamp hoặc @UpdateTimestamp

DATE                  LocalDate            java.time.LocalDate            Chỉ lưu Ngày-Tháng-Năm

ENUM                  Enum (Tự tạo)        (Không có)                     Bắt buộc thêm: @Enumerated(EnumType.STRING) để lưu dạng chữ

JSON                  String / List / Map  java.lang.String               Nếu map ra List/Map, bắt buộc dùng @Convert(converter = MyJsonConverter.class)
=========================================================================================================

Bạn hãy đóng vai một Senior Backend Engineer, thực hiện Auditing chi tiết bộ Entity này dựa trên các tiêu chuẩn sau:

1. Mối quan hệ (Relationships): Kiểm tra tính đối xứng của quan hệ hai chiều. 'mappedBy' đã khớp với tên biến ở đầu @ManyToOne chưa? Có bị thiếu đầu @OneToMany nào cần thiết để truy vấn ngược không?
2. Khởi tạo Collection: Kiểm tra tất cả các Set hoặc List đã được khởi tạo bằng 'new HashSet<>()' hoặc 'new ArrayList<>()' chưa để tránh NullPointerException khi add element.
3. Java Patterns & JPA: 
   - Đã có đầy đủ @NoArgsConstructor (bắt buộc cho JPA) chưa?
   - Đã cài đặt hashCode() và equals() chuẩn chưa (ưu tiên dùng ID, tránh dùng @Data của Lombok gây đệ quy vô tận)?
4. Performance & Logic: 
   - Check FetchType. Các quan hệ @ManyToOne đã để LAZY chưa?
   - Check CascadeType cho các quan hệ cha-con (như Order-OrderDetail).
   - Soi lỗi chính tả (typo) trong tên biến và @Column, @JoinColumn.

Hãy liệt kê chi tiết các lỗi sai theo từng File và hướng dẫn cách sửa cụ thể cho tôi.
